# the-star-way 功能清单与 AI API 依赖

## 1. 分类原则

| 标记 | 含义 |
|---|---|
| 不需要 AI API | 可以依赖 GitHub API、本地 SQLite、规则、前端状态或静态 Demo 完成。 |
| 可选 AI API | 没有 AI 也能运行基础版本；启用 AI 后提升摘要、翻译、推荐或报告质量。 |
| 需要 AI API | 该功能核心价值来自模型生成、理解、总结或推理；没有 AI 只能展示占位或规则降级结果。 |

## 8. 实施优先级建议

1. 先完成不需要 AI API 的 P0：GitHub 同步、SQLite、规则分类、查询、导出。
2. 再补可选 AI API 的增强：主题聚类命名、隐藏宝石解释、相似项目解释、技术人格总结。
3. 最后做强依赖 AI API 的内容生成：README 摘要、Star DNA、学习路径、复用建议、仓库简介翻译。

## 9. 当前结论

首版 MVP 不应依赖 AI API 才能运行。AI API 应作为增强能力接入，核心浏览、同步、筛选、规则分类、统计和导出必须保持本地可用。
